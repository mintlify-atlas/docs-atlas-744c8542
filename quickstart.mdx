Huashu Design is a skill you install into your agent once and then drive entirely through conversation. After a single `npx` command you can ask for a clickable app prototype, an animated product launch video, or an editable slide deck — and receive a finished file in minutes, no design tool required.

## Install the skill

<Steps>
  <Step title="Run the install command">
    Open your terminal and install the skill with the `skills` CLI:

    <CodeGroup>

    ```bash npx
    npx skills add alchaincyf/huashu-design
    ```

    </CodeGroup>

    This fetches `SKILL.md` and the supporting reference files from the `alchaincyf/huashu-design` repository and registers them with your agent.
  </Step>

  <Step title="Verify installation">
    Ask your agent a quick sanity-check question:

    ```text
    What design capabilities does the Huashu Design skill give you?
    ```

    Your agent should describe the seven capability categories — interactive prototypes, slide decks, motion design, design variations, infographics, design direction advisor, and expert critique. If it lists them, the skill is active.
  </Step>

  <Step title="Give your first prompt">
    Start with a concrete task. The skill works best when you describe the deliverable you want, not the steps to get there:

    ```text
    Build an iOS prototype for a Pomodoro app — 4 screens, actually clickable.
    ```

    The agent will ask a focused batch of clarifying questions, then begin building.
  </Step>
</Steps>

## Supported agents

Huashu Design is agent-agnostic. Install it into any agent that supports markdown skills:

- Claude Code
- Cursor
- Codex
- OpenClaw
- Hermes
- Trae

<Note>
  The skill's internal prompts (`SKILL.md` and the `references/` files) are written in Chinese, but the agent handles English tasks without issue. All demos in this documentation use English prompts.
</Note>

## First design walkthrough: Pomodoro app prototype

The iOS prototype task is a good first run because it exercises the skill's core workflow end-to-end. Here is what happens after you send the prompt:

<Steps>
  <Step title="Clarifying questions">
    The agent sends a single batch of questions — preferred color scheme, number of screens, whether you want real images or placeholders, and so on. Answer them all at once before the agent begins building.
  </Step>

  <Step title="Skeleton with assumptions">
    The agent produces a first HTML file with placeholder blocks, assumption comments inline, and a reasoning section at the top. This is intentional: showing you a rough draft early costs nothing to correct; correcting a finished prototype is expensive.
  </Step>

  <Step title="Real content pass">
    After you confirm the structure, the agent fills in real images (pulled from Wikimedia, Met, or Unsplash), interactive state management across screens, and the pixel-accurate iPhone 15 Pro bezel complete with Dynamic Island, status bar, and Home Indicator.
  </Step>

  <Step title="Playwright verification">
    Before delivering the file, the agent runs Playwright click tests to confirm that every navigation tap works as expected.
  </Step>

  <Step title="Delivery">
    You receive a single self-contained HTML file. Open it in any browser to interact with the prototype.
  </Step>
</Steps>

<Tip>
  If your brief is vague — "make something nice" or "design an app" — the skill enters Design Direction Advisor mode automatically. It recommends three differentiated visual directions drawn from five design schools and twenty philosophies, generates a demo for each in parallel, and lets you choose before doing any real work.
</Tip>
